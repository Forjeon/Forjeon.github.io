SBRES


The 'Script Based Record Entry System', or 'SBRES', is a command-line program that saves user entries as webpages. The user may enter a text paragraph or the location and caption to an image on each entry. Once finished, the entry is written to an HTML page and the index page is updated with a link to the new entry. I built this to help me keep a diary in a more fun way, and also to help me with making and keeping a legally accurate engineering journal. Sadly, this was built in Python 2.7, and has since been broken by updates to Python code.

Python Driftcar Game


This is the very first 'game' I ever built. It was also my first foray into game engines and programming other than making Minecraft mods. It is built in the Blender Game Engine. As Blender uses python, so did I in this simple game. The player is a green triangular 'car'; they drive around and drift on an enclosed, simple board. I was eventually going to add AI competetive racers, but as I explored the world of Game Development I moved on to other, larger projects and forgot about this one. Later, after my hard drive became corrupted, I could only recover fragments of the game in early development, not the finished 'demo' I had made and textured.

BGE Metroidvania Platformer


This is a simple demo of a game I was working on sometime 2016-2017. I've since stopped work on the project, and lost many of the files to a corrupted hard drive a few years back. The game was built only using Python code. I used Blender's BGE (pre Blender 2.8) and the Bullet physics engine. The demo runs on only about 10 python scripts, most of which control the various actions of the user player. The game is in the Metroidvania style, and the demo features a small bit of platforming to test and perfect the responsiveness of the player's jump. An AI enemy at the top of the platforming section patrols a small rectangular area. The player can run, jump, double jump, and fire purple energy bolts from their still unmodeled gun.

VR Handtracking Pose-Check System


In this game environment I experiment with the accuracy and tactile-control that hand-tracked VR apps can offer. The system can recognize any hand-finger pose and--after a calibration phase--recall and identify said poses at any later time. A 'system alert' gesture is used to tell the system to look for the poses; this saves on processing power by only running the pose-checker when the user intends to activate a pose. I also built the custom scipts that control the playing, looping, and changing of sound effects, as well as the scripts controlling the firing, charging, projectile settings, and projectile interactions of the user's arm cannon. This game also features IK rigging on the player avatar, allowing arms and a full body to be visible without adding in additional tracking hardware / software.
