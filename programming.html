VR Hand-tracking Gesture Tracker


This is a c# script I wrote that pulls rotation and position info from Oculus' OVRSkeleton system for Quest hand-tracking to gather various stats. I show off all basic functions of this 'gesture tracker' script in the above video, though the system has many more complex actions that are hard to show through video and text; those are not seen in the video.

The system is built to consume as little processing and memory as possible, as it is specifically designed to run on the Oculus Quest mobile VR instead of a PC based setup. All returns from the script are only calculated when the function is called, and all output is saved on the stack for quick access. The only exception to this is the tracking of 'old' position and rotation data, used to calculate linear and angular velocity when the functions are called.

The script:
— is aware of the exact physical dimensions of the user's hands
— proportionally calculates the range in which any user, given their hands' dimensions, is making a fist
— can express the certanty of the user making a fist as a percentage for each finger
— can remember and identify any number of finger poses (up to the limit of storing 3d float coords on a mobile VR device) by tracking the exact location of the distal bone of the user (relative to the entire hand)
— can recognize Oculus-specific gestures such as system gestures (exit to menu, recenter, etc.) and pinching
— can return the exact position and rotation of any hand (using the wrist) or any finger (using the distal bone)
— can return the approximate (~3 decimal place accuracy) linear and angular velocity of any hand (using the wrist) or any finger (using the distal bone)
— can return the exact position of finger tips—useful for touch interfaces ie. buttons, virtual keyboards

SBRES


The 'Script Based Record Entry System', or 'SBRES', is a command-line program that saves user entries as webpages. The user may enter a text paragraph or the location and caption to an image on each entry. Once finished, the entry is written to an HTML page and the index page is updated with a link to the new entry. I built this to help me keep a diary in a more fun way, and also to help me with making and keeping a legally accurate engineering journal. Sadly, this was built in Python 2.7, and has since been broken by updates to Python code.

Python Driftcar Game


This is the very first 'game' I ever built. It was also my first foray into game engines and programming other than making Minecraft mods. It is built in the Blender Game Engine. As Blender uses python, so did I in this simple game. The player is a green triangular 'car'; they drive around and drift on an enclosed, simple board. BGE Metroidvania Platformer


This is a simple demo of a game I was working on sometime 2016-2017. I've since stopped work on the project, and lost many of the files to a corrupted hard drive a few years back. The game was built only using Python code. I used Blender's BGE (pre Blender 2.8) and the Bullet physics engine. The demo runs on only about 10 python scripts, most of which control the various actions of the user player. The game is in the Metroidvania style, and the demo features a small bit of platforming to test and perfect the responsiveness of the player's jump. An AI enemy at the top of the platforming section patrols a small rectangular area. The player can run, jump, double jump, and fire purple energy bolts from their still unmodeled gun.
